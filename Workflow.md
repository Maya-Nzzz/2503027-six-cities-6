# Как работать над проектом

## Окружение

Для удобства работы над проектом используются инструменты из **Node.js** и **npm**. Все необходимые настройки произведены. Убедитесь, что на рабочем компьютере установлен актуальный LTS релиз Node.js**. Актуальная версия **Node.js** указана в файле `package.json` в поле `node`. Затем, в терминале, перейдите в директорию с проектом и _единожды_ запустите команду:

```bash
npm install
```

Команда запустит процесс установки зависимостей проекта из **npm**.

### Переменные окружения

Ниже пример всех переменных окружения в формате `PARAM=value — описание`.

- PORT=5000 — порт для входящих соединений.
- SALT=secret — соль для хеширования паролей.
- DB_HOST=127.0.0.1 — адрес сервера базы данных (MongoDB).
- DB_PORT=27017 — порт подключения к MongoDB.
- DB_USER=admin — пользователь для подключения к базе данных.
- DB_PASSWORD=test — пароль для подключения к базе данных.
- DB_NAME=six-cities — имя базы данных.
- UPLOAD_DIRECTORY=upload — директория для сохранения загружаемых файлов.
- JWT_SECRET=secret — секретная строка для подписи JWT.


### Запуск проекта

1. Установите актуальную LTS-версию Node.js.
2. Выполните установку зависимостей:

```bash
npm install
```

3. Создайте в корне проекта файл `.env` на основе `.env.example` и при необходимости скорректируйте значения переменных окружения.
4. Запустите MongoDB. Если используется Docker, выполните:

```bash
docker compose up -d
```

5. Запустите сервис в режиме разработки:

```bash
npm run start:dev
```

6. Для сборки и запуска production-версии используйте:

```bash
npm start
```

Если требуется отдельно проверить CLI, выполните:

```bash
npm run cli -- <команда>
```


### Сценарии

В `package.json` предопределено несколько сценариев.

- `npm start` — собирает проект и запускает REST-сервис из `dist/main.rest.js`.
- `npm run start:dev` — запускает сервис в режиме разработки через `nodemon`.
- `npm run build` — выполняет полную сборку: очищает `dist` и компилирует проект заново.
- `npm run clean` — удаляет директорию `dist`.
- `npm run compile` — компилирует TypeScript-проект в `dist`.
- `npm run lint` — запускает ESLint для файлов в `src`.
- `npm run cli -- <команда>` — запускает CLI-приложение.
- `npm run mock:server` — запускает mock-сервер с тестовыми данными через `json-server`.

#### Скомпилировать проект

```bash
npm run compile
```

Создаст директорию `dist` и скомпилирует проект.

#### Удалить скомпилированный проект

```bash
npm run clean
```

Удаляет директорию `dist`. Используется перед компиляцией.

#### Собрать проект

```bash
npm run build
```

Выполняет сборку проекта: удаляет ранее скомпилированный проект и компилирует заново.

#### Проверить линтером

```bash
npm run lint
```

Запуск проверки проекта статическим анализатором кода **ESLint**.

Линтер проверяет файлы только внутри директории `src`.

**Обратите внимание**, при запуске данной команды, ошибки выводятся в терминал.

#### Запустить проект

```bash
npm start
```

#### Запустить проект в режиме разработки

```bash
npm run start:dev
```

#### Запустить cli приложение

```bash
npm run cli -- <команда>
```

Команды:
- `--version` - выводит номер версии
- `--help` - печатает этот текст
- `--import <path>` - импортирует данные из TSV
- `--import <path> <db-uri> <salt>` - импортирует данные из TSV в MongoDB
- `--import <path> <user> <password> <host> <db> <salt>` - импортирует данные из TSV в MongoDB (параметры подключения отдельно)
- `--generate <n> <path> <url>` - генерирует произвольное количество тестовых данных

#### Запустить mock-сервер

```bash
npm run mock:server
```

Запускает локальный mock-сервер с тестовыми данными. Используется для разработки и проверки сценариев генерации данных.
